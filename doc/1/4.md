# File Organization For WebGL programs

Как мы обсуждали ранее, типичная программа WebGL состоит из следующих типов файлов:

* HTML-файл, содержащий описание веб-страницы.
* Один или несколько файлов CSS, описывающих свойства элементов HTML на веб-странице.
* Один или несколько файлов JavaScript, которые обеспечивают логику обработки для программы WebGL.
* Один или несколько файлов данных, описывающих объекты, которые будут отображаться.
* Один или несколько файлов шейдеров, которые являются программами рендеринга для графического конвейера. 

Нам нужна файловая структура, которая поможет управлять всеми этими файлами. Но на самом деле это более сложно, чем просто управлять одной программой WebGL. Веб-сайт, содержащий компьютерную графику WebGL, может иметь несколько страниц, содержащих графику, и каждая веб-страница может содержать несколько программ WebGL, которые визуализируются в отдельных областях холста. Если все ваши файлы хранятся в одной папке на вашем сервере, вам будет очень сложно управлять этими файлами, поскольку вы будете выполнять обновления и улучшения с течением времени.

Основной принцип всех файловых структур - сгруппировать связанные файлы в отдельную папку. Если у вас есть несколько программ WebGL для веб-сайта, некоторые файлы, скорее всего, будут совместно использоваться программами. Следовательно, логическая файловая организация должна иметь **отдельную папку для каждой программы WebGL**, в которой хранятся файлы, уникальные для этой программы, и общую папку библиотеки для общих файлов.

## File Organization Used for These Tutorials

Следующая файловая структура обеспечивает базовую организацию файлов, необходимых для веб-сайта, реализующего несколько программ WebGL. Имена папок соответствуют содержимому папки. Обратите внимание, что:

* Все файлы, уникальные для программы WebGL, сохраняются в уникальной папке для этой программы.
* Все файлы, совместно используемые программами WebGL, хранятся отдельно в папке общей библиотеки под названием lib.
* Папка «Общая библиотека» организована по типам файлов.

Хотя эта файловая структура сложна, она лучше, чем ее отсутствие. Если программа WebGL не работает, первое, что вы должны проверить, - все ли необходимые файлы загружаются правильно. Вы можете просмотреть ошибки, связанные с отсутствующими файлами, в окне консоли JavaScript. Более подробную информацию об окне консоли можно найти в отладочной информации в Разделе 2.
