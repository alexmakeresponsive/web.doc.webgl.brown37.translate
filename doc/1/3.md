# Computer Graphics - A Brief History

Первая система проектирования компьютерной графики была разработана **Эваном Сазерлендом** в качестве его кандидатской диссертации в Массачусетском технологическом институте в 1963 году. Она называлась **Sketchpad** и позволяла пользователю рисовать механическую деталь на экране компьютера, накладывать ограничения на деталь и заставлять компьютер вычислять точный дизайн детали. Это было революционно, и с тех пор компьютерная графика стала популярной.

В первые дни, если вы хотели создавать компьютерную графику, вам приходилось писать программы, которые напрямую взаимодействовали с графическим оборудованием. Поскольку были созданы новые конструкции оборудования, программное обеспечение пришлось полностью переписать для работы с новым оборудованием. На протяжении 1970-х годов было создано множество «графических библиотек» для решения этой проблемы, но большинство библиотек работали с отдельными операционными системами, и их было нелегко перенести на новые операционные системы или новое оборудование. В 1980-х годах ISO (Международная организация по стандартизации) попыталась создать стандартную библиотеку компьютерной графики под названием **PHIGS**. Было заявлено, что это лучший и окончательный ответ для создания компьютерной графики на всех вычислительных платформах. Однако это был полный провал по одной простой причине: PHIGS не только предоставил библиотеку функций для рисования трехмерной графики, но и точно указывал, как вы должны хранить и организовывать свои графические данные. Программисты это ненавидели.

## GL and OpenGL

Первым производителем компьютерной графики в конце 1980-х и на протяжении 1990-х годов была компания Silicon Graphics, Inc. Она была производителем оборудования, но поставляла свои компьютеры с проприетарным программным интерфейсом приложений компьютерной графики (API), известным как язык графики IRIS (IRIS GL). , Тогдашние программисты искали альтернативу PHIGS, и в 1992 году Silicon Graphics выпустила свою графическую библиотеку (GL) как «открытый» стандарт под названием OpenGL. Цитируя Википедию: 

> «Это означало, что впервые можно было писать быстрые, эффективные кроссплатформенные графические программы. По сей день OpenGL остается единственным стандартом трехмерной графики в реальном времени, который можно переносить в различные операционные системы. OpenGL-ES работает даже на многих типах сотовых телефонов. Его главный конкурент (Direct3D от Microsoft) работает только на компьютерах с Microsoft Windows и некоторых консолях ».

OpenGL кардинально изменился с годами. Здесь вы можете увидеть подробную информацию о версии. Основные изменения между версиями перечислены ниже для 
краткости.

* 1992: OpenGL 1.0 - конвейер с фиксированными функциями для создания компьютерной графики. Программист отправил данные вершин в конвейер, и на выходе получился 3D-рендеринг.
* 2004: OpenGL 2.0 - GPU стал программируемым. Программисты могли писать шейдерные программы, скомпилированные в аппаратное обеспечение. Программы конвейера с фиксированными функциями и программы шейдеров несовместимы.
* 2008: OpenGL 3.0 - **графические данные были перемещены из оперативной памяти ЦП в память ГП** для более быстрого рендеринга.
* 2010: OpenGL 4.0 - Расширенные возможности языка шейдеров и расширенное наложение текстур.
* 2014: OpenGL 4.5 - самая последняя версия.

## OpenGL ES

OpenGL - это спецификация оборудования, для работы которой требуется довольно мощное оборудование. Программные драйверы для API очень большие и сложные. Поскольку на рынке начали появляться мобильные устройства с низким энергопотреблением, для этих устройств требовалось решение с трехмерной графикой. Таким образом был создан OpenGL ES (Embedded Subset). С веб-сайта opengl OpenGL ES «сокращает довольно большой API OpenGL до самого необходимого, чтобы его можно было реализовать на устройствах с более простым и дешевым оборудованием и, прежде всего, с достаточно низкими требованиями к мощности для работы от батарей. Например, он доступен в стандартной комплектации на смартфонах под управлением операционной системы Apple IOS и Google Android ». OpenGL ES непрерывно развивался от версии 1.0 в 2003 году до самой последней версии 3.2. Полную информацию можно найти здесь: [wikipedia](https://en.wikipedia.org/wiki/OpenGL_ES).

## WebGL

Разработчики программного обеспечения постоянно искали способы «написать один раз, работать где угодно». Это был девиз языка **Java, созданный Sun Microsystems** в конце 1990-х годов. Если разработчики программного обеспечения могут писать программы, работающие на нескольких платформах, они могут получить большую прибыль от своих усилий. Для некоторых веб-браузер является идеальной целью «написать один раз, запустить где угодно», особенно с выпуском HTML версии 5.0. Дополнительные функции HTML позволяют писать сложные, мощные и кроссплатформенные веб-приложения. Обеспечение **доступа к графическому процессору (GPU)** в браузере стало реальностью с созданием WebGL.

Что такое WebGL? Со страницы WebGL Wikipedia: 

> «WebGL (библиотека веб-графики) - это API JavaScript для рендеринга интерактивной компьютерной 3D-графики и 2D-графики в любом совместимом веб-браузере без использования подключаемых модулей. WebGL полностью интегрирован во все веб-стандарты браузера, что позволяет с ускорением на GPU использовать физику, обработку изображений и эффекты как часть холста веб-страницы. Элементы WebGL можно смешивать с другими элементами HTML и комбинировать с другими частями страницы или фоном страницы. Программы WebGL состоят из управляющего кода, написанного на JavaScript, и кода шейдера, который выполняется на графическом процессоре (GPU) компьютера ».

**WebGL - это OpenGL ES 2.0, реализованный для любого веб-браузера, совместимого с HTML 5.0.**

Поскольку все веб-браузеры поддерживают JavaScript, WebGL определяет API JavaScript для рендеринга трехмерной графики в реальном времени. А поскольку практически все устройства сегодня имеют графический процессор, это означает, что вы можете написать одну программу, которая будет выполняться практически на всех существующих сегодня - и завтра - компьютерных устройствах благодаря стандартам HTML 5.0 и WebGL.

Так что же такое WebGL? Это будущее трехмерной компьютерной графики в реальном времени.
