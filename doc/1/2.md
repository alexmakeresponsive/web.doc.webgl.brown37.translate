# 3D Computer Graphics - What and How

Компьютерная графика - это изображения и фильмы, созданные с помощью компьютеров. (1)

Есть два основных способа описать картинку с помощью компьютера:

* Растровая графика - описывает изображение с использованием множества маленьких цветных точек. Каждая точка называется пикселем, что является аббревиатурой от «элемента изображения». Если точки достаточно маленькие и расположены достаточно близко друг к другу, человек не видит отдельные точки, а скорее видит «картинку».
* Векторная графика - описывает объекты как геометрические фигуры с помощью математических уравнений. Изображение создается из математических описаний с помощью процесса, называемого рендерингом. Результатом рендеринга является двухмерное растровое изображение.

Компьютерные инструменты были разработаны для создания и управления обоими типами компьютерной графики. Например, Adobe PhotoShop - лучший инструмент для работы с растровыми изображениями. Но создание новых изображений путем манипулирования множеством маленьких цветных точек невыгодно для движущихся изображений, где объекты меняются с течением времени. PhotoShop отлично подходит для создания отдельных изображений или «подправки» существующего изображения, но он не подходит для создания фильмов.

**Computer-generated imagery (CGI)** - это применение компьютерной графики для создания или использования изображений в искусстве, печатных СМИ, видеоиграх, фильмах, телевизионных программах, рекламных роликах, видео и симуляторах. (2) Обычный человек, вероятно, слышал термин CGI только в отношении видеоигр и фильмов, но он имеет широкое применение во многих областях.

3D компьютерная графика - это графика, в которой используется трехмерное представление геометрических данных для выполнения вычислений и визуализации 2D-изображений. (3) Эти учебные пособия помогут вам узнать, как использовать методы компьютерной 3D-графики для создания CGI. Или, говоря другими словами, эти учебники помогут вам создать векторные графические представления трехмерных объектов, а затем преобразовать их в растровые изображения. Если вы можете создавать данные и алгоритмы, которые визуализируют растровые изображения менее чем за 1/30 секунды, вы можете создавать видео в реальном времени, которое отображается, пока зритель смотрит. Большинство видеоигр основано на **рендеринге в реальном времени**. Если создание каждого изображения занимает больше 1/30 секунды, вы всегда можете сохранить изображения и воспроизвести их в режиме реального времени. Большинство фильмов CGI не отображается в реальном времени.
 
Так как же векторные графические изображения объектов преобразуются в растровое изображение? Это нетривиальный процесс, требующий нескольких шагов. Эти шаги хорошо изучены в течение многих лет, и они должны выполняться в четко определенной последовательности. Эта последовательность операций называется **«графическим конвейером»**. В первые дни компьютерной графики «конвейер» не был программируемым. Данные подавались в конвейер, и на выходе получалось растровое изображение. Но современные графические процессоры позволяют программисту управлять определенными частями конвейера с помощью **шейдерных программ**. Программа шейдера - это просто очень специализированный набор инструкций для управления графическими данными на критических этапах конвейерного процесса. Шейдерные программы предоставляют программисту удивительную гибкость и творческий потенциал, но за счет дополнительной сложности. Цель этих руководств - помочь вам понять графический конвейер и способы управления графическими данными с помощью программ шейдеров.

## The Graphics Pipeline
 
| Pipeline Step        | Description           | Performed by  |
| ------------- | ------------- | ----- |
| **INPUT** | Подавайте данные в конвейер, включая вершины модели (x, y, z), которые определяют местоположение, векторы нормалей (dx, dy, dz), которые определяют направление, и данные цвета. | CPU: JavaScript code |
| **MODEL VIEW TRANSFORM** | Перемещайте, масштабируйте и поворачивайте модели в желаемое положение и ориентацию в 3D-сцене. Затем переместите все перед камерой. | CPU: JavaScript code <br> GPU: Vertex Shader |
| **PROJECTION TRANSFORM** | Спроецируйте трехмерный мир на экран просмотра 2D. | CPU: JavaScript code <br> GPU: Vertex Shader |
| **NORMALIZE TRANSFORM** <br> **CLIP TRANSFORM** | Отрежьте все, что находится вне поля зрения камеры. | CPU: JavaScript code <br> GPU: Vertex Shader |
| **VIEWPORT TRANSFORM** | Сопоставьте координаты 3D-объекта с пиксельными координатами растрового изображения. | GPU: Fixed functionality |
| **RASTERIZE** | Определите, какие пиксели покрыты каждым объектом (точкой, линией или треугольником), и выбросьте объекты, скрытые от просмотра другими объектами. | GPU: Fixed functionality | 
| **SHADING** | Определите цвет каждого пикселя, представляющего объект. | CPU: JavaScript code <br> GPU: Fragment Shader | 
| **COMPOSITING** | Добавьте цвет пикселя к растровому изображению, возможно, комбинируя цвет с цветом, который уже присутствует в изображении. | GPU: Fixed functionality | 
| **OUTPUT** | Выведите растровое изображение, одно значение цвета для каждого пикселя изображения. | GPU: Fixed functionality |

Подробности каждого из вышеперечисленных шагов объяснены в остальных уроках. По мере того, как вы погружаетесь в детали, часто возвращайтесь к этому обзору, чтобы иметь в виду общую картину.
